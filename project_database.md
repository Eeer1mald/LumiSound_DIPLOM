# 2.5 Защита и сохранность данных

Данные хранятся в облачном сервисе Supabase, который предоставляет управляемую базу данных PostgreSQL с встроенными механизмами аутентификации, хранилищем файлов и REST API.

Ограничение доступа к данным реализовано на двух уровнях. На уровне базы данных для каждой таблицы включена политика Row Level Security (RLS), которая гарантирует, что пользователь может читать и изменять только собственные данные. На уровне приложения доступ к приватным операциям осуществляется исключительно через авторизованные запросы с передачей Bearer-токена, получаемого при входе в аккаунт. Публичные запросы (например, просмотр рецензий других пользователей) выполняются через anon-ключ с ограниченными правами.

Для авторизации пользователю необходимо заполнить экран авторизации в соответствии с рисунком 2.16.

# 2.6 Организация и ведение информационной базы (модели)

Все данные проекта хранятся в облачном сервисе Supabase на основе реляционной базы данных PostgreSQL. Взаимодействие с базой данных осуществляется через REST API (PostgREST), предоставляемый Supabase.

В используемой базе данных хранятся следующие таблицы: профили пользователей, оценки треков, комментарии к трекам, избранные треки, избранные исполнители, история прослушиваний, плейлисты, треки в плейлистах, лайки плейлистов, сессии синтеза и участники синтеза. Основные из них представлены в виде таблиц 2.1 – 2.8.
