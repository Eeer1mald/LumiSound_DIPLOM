# 📖 ПОШАГОВАЯ ИНСТРУКЦИЯ: Как создать таблицу profiles в Supabase

## 🎯 Цель
Создать таблицу `profiles` в вашей базе данных Supabase, чтобы приложение могло сохранять информацию о профилях пользователей (имя, биография, любимый жанр, аватар).

---

## 📝 ШАГ 1: Откройте SQL Editor в Supabase

1. Откройте браузер и перейдите на сайт **https://supabase.com**
2. Войдите в свой аккаунт (если еще не вошли)
3. Найдите ваш проект **LumiSound** в списке проектов и нажмите на него
4. В левом боковом меню найдите раздел **"SQL Editor"** и нажмите на него
   - Это тот же раздел, где вы видите ваш текущий SQL с таблицами users, tracks, ratings

---

## 📝 ШАГ 2: Откройте файл с SQL-скриптом

1. В корне вашего проекта найдите файл: **`create_profiles_table_final.sql`**
2. Откройте этот файл в любом текстовом редакторе (Notepad++, VS Code, или даже обычный Блокнот)
3. Выделите **ВЕСЬ** текст в файле (Ctrl+A) и скопируйте его (Ctrl+C)

---

## 📝 ШАГ 3: Вставьте SQL-скрипт в редактор

1. Вернитесь в SQL Editor в Supabase
2. В центральной части экрана вы увидите большое текстовое поле (это и есть редактор SQL)
3. **Вариант А:** Если в редакторе уже есть какой-то код, вы можете:
   - Создать новую вкладку (обычно кнопка "+" или "New Query")
   - Или удалить весь существующий текст (Ctrl+A, затем Delete)
4. Вставьте скопированный SQL-скрипт в редактор (Ctrl+V)

---

## 📝 ШАГ 4: Проверьте SQL-скрипт перед выполнением

После вставки вы должны увидеть примерно такой текст:

```sql
-- ============================================================
-- СОЗДАНИЕ ТАБЛИЦЫ PROFILES ДЛЯ ПРИЛОЖЕНИЯ LUMISOUND
-- ============================================================
...
CREATE TABLE IF NOT EXISTS public.profiles (
    id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
    username TEXT NOT NULL,
    email TEXT NOT NULL UNIQUE,
    ...
```

**Важно:** Весь скрипт должен быть вставлен целиком - от первого комментария до последнего.

---

## 📝 ШАГ 5: Выполните SQL-скрипт

1. Внизу страницы найдите кнопку **"Run"** или **"Execute"** или **"Run CTRL ↵"** (большая зеленая кнопка)
2. Нажмите на эту кнопку
3. Подождите несколько секунд - скрипт выполняется

---

## 📝 ШАГ 6: Проверьте результат

После выполнения скрипта вы должны увидеть:

### ✅ УСПЕШНЫЙ РЕЗУЛЬТАТ:
В нижней панели (вкладка "Results") вы увидите что-то вроде:
- "Success" (Успешно)
- Или список созданных объектов (TABLE, INDEX, POLICY, TRIGGER и т.д.)

### ❌ ЕСЛИ БЫЛА ОШИБКА:
Вы увидите красное сообщение с описанием ошибки. Частые ошибки:

1. **"relation already exists"** - таблица уже существует
   - **Решение:** Это нормально! Скрипт использует `IF NOT EXISTS`, так что ничего страшного

2. **"permission denied"** - нет прав доступа
   - **Решение:** Убедитесь, что вы используете роль администратора (обычно "Role: postgres" внизу справа)

3. **"syntax error"** - ошибка синтаксиса
   - **Решение:** Проверьте, что вы скопировали весь скрипт полностью, без пропусков

---

## 📝 ШАГ 7: Проверьте, что таблица создана

1. В левом меню Supabase найдите раздел **"Table Editor"** (Редактор таблиц)
2. В списке таблиц вы должны увидеть новую таблицу **`profiles`**
3. Нажмите на нее, чтобы увидеть структуру:
   - Должны быть колонки: id, username, email, bio, favorite_genre, avatar_url, created_at, updated_at

## ❓ ЧАСТЫЕ ВОПРОСЫ

**В: Что будет, если я выполню скрипт два раза?**
О: Ничего страшного! Скрипт использует `IF NOT EXISTS` и `DROP IF EXISTS`, так что он безопасен для повторного выполнения.

**В: Могу ли я изменить структуру таблицы после создания?**
О: Да, но для этого нужно будет писать дополнительные SQL-запросы (`ALTER TABLE`). Пока этого не требуется.

**В: Что если у меня возникла ошибка?**
О: Скопируйте текст ошибки и сообщите мне. Я помогу исправить.
