# Инструкция по настройке загрузки аватаров в Supabase

## Что было реализовано

1. ✅ Загрузка обрезанного изображения в Supabase Storage
2. ✅ Сохранение URL аватара в таблицу `profiles` (поле `avatar_url`)
3. ✅ Автоматическая загрузка аватара из Supabase при открытии профиля
4. ✅ Сжатие изображения перед загрузкой (максимум 512x512px, до 1MB)

## Настройка Supabase Storage

### Шаг 1: Создание bucket "avatars"

1. Откройте [Supabase Dashboard](https://supabase.com/dashboard)
2. Выберите ваш проект
3. Перейдите в раздел **SQL Editor** (в левом меню)
4. Откройте файл `setup_avatars_storage.sql` из корня проекта
5. Скопируйте весь скрипт и вставьте в SQL Editor
6. Нажмите **Run** или **Execute**

Этот скрипт:
- Создаст bucket "avatars" (если его еще нет)
- Настроит bucket как публичный (для доступа к аватарам без авторизации)
- Установит лимит размера файла: 5MB
- Разрешит форматы: JPEG, JPG, PNG, WebP
- Настроит политики безопасности (RLS)

### Шаг 2: Проверка настройки

После выполнения скрипта:

1. Перейдите в раздел **Storage** в Supabase Dashboard
2. Убедитесь, что bucket "avatars" создан и помечен как **Public**
3. Проверьте, что политики безопасности применены

## Как это работает

### Процесс загрузки аватара:

1. Пользователь выбирает изображение из галереи
2. Изображение обрезается через uCrop
3. Изображение сжимается до 512x512px и максимум 1MB
4. Файл загружается в Supabase Storage (bucket "avatars")
5. Получается публичный URL вида: `{baseUrl}/storage/v1/object/public/avatars/{fileName}`
6. URL сохраняется в таблицу `profiles` (поле `avatar_url`)
7. Аватар сразу отображается в приложении

### Процесс загрузки при открытии профиля:

1. При открытии экрана профиля загружается профиль из таблицы `profiles`
2. Если в профиле есть `avatar_url`, он используется для отображения аватара
3. Coil загружает изображение по URL из Supabase
4. Если изображение не загружается, показывается первая буква имени пользователя

## Формат имени файла

Аватары сохраняются с именами вида:
```
avatar_{userId}_{timestamp}.jpg
```

Например:
```
avatar_123e4567-e89b-12d3-a456-426614174000_1703123456789.jpg
```

## Важные замечания

1. **Bucket должен быть публичным** - иначе изображения не будут доступны без авторизации
2. **Политики безопасности** - любой авторизованный пользователь может загружать аватары, но в приложении контролируется, что пользователь загружает только свой аватар
3. **Лимит размера** - файлы больше 5MB будут отклонены Supabase
4. **Форматы** - поддерживаются только JPEG, JPG, PNG, WebP

## Устранение проблем

### Аватар не загружается в Supabase Storage

1. Проверьте, что bucket "avatars" создан и является публичным
2. Проверьте политики безопасности в разделе Storage → Policies
3. Проверьте логи в Android Studio (Logcat) - ищите теги "ProfileViewModel" и "SupabaseService"

### Аватар не отображается в приложении

1. Проверьте, что URL сохранен в таблице `profiles` (поле `avatar_url`)
2. Проверьте, что URL доступен в браузере (откройте URL в браузере)
3. Проверьте интернет-соединение
4. Проверьте логи в Android Studio

### Ошибка "Не удалось загрузить аватар"

1. Проверьте размер файла (должен быть меньше 5MB)
2. Проверьте формат файла (должен быть JPEG, JPG, PNG или WebP)
3. Проверьте, что пользователь авторизован (есть accessToken)
4. Проверьте логи в Android Studio для деталей ошибки

## Технические детали

- **Storage API endpoint**: `/storage/v1/object/avatars/{fileName}`
- **Публичный URL**: `{baseUrl}/storage/v1/object/public/avatars/{fileName}`
- **Метод загрузки**: POST с заголовком `x-upsert: true` (перезаписывает существующие файлы)
- **Content-Type**: `image/jpeg`
- **Сжатие**: до 512x512px, качество JPEG настраивается автоматически (90% → 30% если нужно)
