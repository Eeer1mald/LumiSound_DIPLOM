# Часто задаваемые вопросы (FAQ)

## 🎨 Дизайн

### Почему используется темная тема?
Темная тема создаёт премиальную атмосферу вечернего прослушивания музыки и снижает нагрузку на глаза при длительном использовании приложения.

### Можно ли изменить цветовую схему?
Да, все цвета определены в дизайн-системе (`/DESIGN_SYSTEM.md`). Вы можете изменить переменные цветов в соответствии с вашими предпочтениями.

### Почему размер экрана 360×800px?
Это стандартный размер для Android-устройств среднего класса, обеспечивающий совместимость с большинством современных смартфонов.

## 🎵 Функциональность

### Как работает система оценок?
Пользователи могут оценивать треки по шкале от 1 до 10 прямо на экране воспроизведения. Эти оценки используются для персонализации рекомендаций.

### Где хранятся оценки пользователей?
В текущей демо-версии оценки хранятся локально. Для продакшн-версии потребуется интеграция с базой данных (см. `/DEVELOPER_GUIDE.md`).

### Как добавить реальное воспроизведение аудио?
Следуйте инструкциям в `/DEVELOPER_GUIDE.md` раздел "Добавление аудио плеера".

## 🔧 Техническое

### Почему используется Tailwind CSS v4?
Tailwind CSS v4 предоставляет современные возможности стилизации с улучшенной производительностью и простотой использования.

### Можно ли использовать приложение без Supabase?
Да, приложение работает с моковыми данными. Supabase нужен только для продакшн-версии с реальной базой данных.

### Как добавить новый экран?
Следуйте шаблону в `/DEVELOPER_GUIDE.md` раздел "Создание нового экрана".

## 📱 Навигация

### Как переключаться между экранами?
Используйте нижнюю навигацию (Bottom Navigation) для перехода между основными экранами: Домой, Поиск, Плейлисты, Профиль.

### Как вернуться назад с экрана воспроизведения?
Нажмите на иконку стрелки вниз (chevron down) в левом верхнем углу экрана.

### Почему экран оценок не в нижней навигации?
Экран оценок доступен через профиль как часть аналитики пользователя, чтобы не перегружать основную навигацию.

## 🗂️ Структура проекта

### Где находятся моковые данные?
В файле `/utils/mockData.ts`. Там хранятся все треки, плейлисты и статистика пользователя.

### Где экраны авторизации?
Экраны Welcome, Login и SignUp находятся в папке `/components/auth/`. Используйте `AuthScreens.tsx` для их отображения.

### Как посмотреть все экраны одновременно?
Импортируйте `AllScreensDemo` в `App.tsx` (см. `/USAGE.md`).

## 🎯 Интеграция

### Нужна ли база данных для работы приложения?
Нет, приложение работает с моковыми данными. База данных нужна только для хранения реальных данных пользователей.

### Какую базу данных использовать?
Рекомендуется Supabase (PostgreSQL) для простоты интеграции. Также подходят Firebase, MongoDB или любая другая БД.

### Как добавить API интеграцию?
Следуйте инструкциям в `/DEVELOPER_GUIDE.md` раздел "Интеграция с бэкендом".

## 🚀 Развертывание

### Где можно разместить приложение?
- Vercel (рекомендуется)
- Netlify
- GitHub Pages
- Любой хостинг для статических сайтов

### Нужен ли сервер?
Нет, это Single Page Application (SPA). Сервер нужен только для API, если вы добавите бэкенд.

### Как оптимизировать для production?
```bash
npm run build
```
Это создаст оптимизированную версию в папке `build/`.

## 🐛 Устранение неполадок

### Изображения не загружаются
Проверьте подключение к интернету. Изображения загружаются с Unsplash через CDN.

### Компонент не отображается
Проверьте импорты и убедитесь, что все необходимые компоненты экспортированы.

### Стили не применяются
Убедитесь, что Tailwind CSS правильно настроен и файл `/styles/globals.css` подключен.

## 💡 Советы

### Как улучшить производительность?
1. Используйте React.memo для компонентов
2. Добавьте виртуализацию для длинных списков (react-window)
3. Оптимизируйте изображения (WebP формат)
4. Используйте код-сплиттинг

### Как добавить анимации?
Используйте библиотеку `framer-motion`:
```bash
npm install framer-motion
```

### Как сделать приложение PWA?
Добавьте `manifest.json` и service worker. Можно использовать `workbox` для упрощения процесса.
