# Логическая и физическая структура приложения LumiSound

## Логическая структура

В состав логической структуры входят следующие компоненты.

Первым компонентом структуры является экран загрузки (этап предзагрузки основных данных и модулей: кэш треков, профиль пользователя, рекомендации и история воспроизведения, необходимые для отображения информации без задержек).

Вторым компонентом структуры является экран приветствия (экран приложения с логотипом и кнопками для перехода к авторизации или регистрации нового пользователя).

Третьим компонентом структуры является экран авторизации (экран приложения с полями для ввода email и пароля пользователя, а также ссылкой на восстановление пароля).

Четвёртым компонентом структуры является экран регистрации (экран приложения с полями для ввода имени пользователя, email, пароля и его подтверждения, с индикатором сложности пароля и валидацией данных).

Пятым компонентом структуры является экран подтверждения email (экран приложения с инструкцией по подтверждению адреса электронной почты после регистрации).

Шестым компонентом структуры является экран настройки профиля (экран приложения с возможностью загрузки и обрезки аватара, вводом никнейма и описания «О себе»).

Седьмым компонентом структуры является главный экран приложения (экран приложения с плейлистами, рекомендованными треками, избранной музыкой и мини-плеером в нижней части экрана).

Восьмым компонентом структуры является экран поиска (экран приложения с поисковой строкой для нахождения треков и исполнителей, а также TikTok-подобным вертикальным фидом с двумя вкладками: «Рекомендации» и «Для вас»).

Девятым компонентом структуры является экран оценок (экран приложения со списком оценённых пользователем треков, фильтрацией по критериям и статистикой оценок по пяти параметрам: Рифмы, Структура, Реализация, Индивидуальность, Атмосфера).

Десятым компонентом структуры является экран профиля пользователя (экран приложения с аватаром, никнеймом, описанием, статистикой активности, а также горизонтальными списками любимых треков и исполнителей).

Одиннадцатым компонентом структуры является экран плеера (полноэкранный экран воспроизведения трека с обложкой в высоком качестве, waveform-прогресс-баром, кнопками управления воспроизведением, плавающими комментариями и возможностью добавления трека в плейлист).

Двенадцатым компонентом структуры является экран исполнителя (экран приложения с аватаром и статистикой исполнителя, биографией, популярными треками и полным списком треков).

Тринадцатым компонентом структуры является экран написания рецензии (экран приложения с полем для ввода текстового комментария к треку длиной до 100 символов).

Четырнадцатым компонентом структуры является экран просмотра рецензий (экран приложения со списком рецензий других пользователей на выбранный трек).

Пятнадцатым компонентом структуры является экран настроек (экран приложения с управлением темой оформления, параметрами воспроизведения, эквалайзером, таймером сна, настройками конфиденциальности и кнопкой выхода из аккаунта).

## Описание потока работы приложения

При запуске приложения первым будет загружаться экран предзагрузки (`AppPreloadViewModel`). В этот момент приложение использует двухфазную стратегию: мгновенно читает данные из локального кэша (DiskCache) и параллельно обновляет их из сети в фоновом режиме.

После завершения предзагрузки приложение проверяет наличие сохранённого токена авторизации. Если пользователь ранее не входил в систему, он переходит на экран приветствия (`WelcomeScreen`), на котором расположены кнопки для перехода к авторизации или регистрации.

При нажатии кнопки «Создать аккаунт» открывается экран регистрации (`RegisterScreen`), где пользователь вводит никнейм, email, пароль и его подтверждение. После успешной регистрации пользователь переходит на экран подтверждения email (`VerifyEmailScreen`), а затем — на экран настройки профиля (`ProfileSetupScreen`), где загружает аватар и заполняет информацию о себе.

При нажатии кнопки «Войти» открывается экран авторизации (`LoginScreen`). После успешного входа пользователь попадает на главный экран приложения (`HomeScreen`).

На главном экране пользователь может просматривать рекомендованные плейлисты и треки, а также переключаться между четырьмя основными разделами через нижнюю навигационную панель: «Главная», «Поиск», «Оценки» и «Профиль».

При нажатии на трек открывается полноэкранный плеер (`PlayerScreen`), в котором отображается обложка трека, waveform-прогресс-бар, кнопки управления воспроизведением (Play/Pause, предыдущий, следующий трек), а также плавающие комментарии других пользователей. Переключение между треками осуществляется горизонтальным свайпом, закрытие плеера — свайпом вниз.

Из экрана плеера пользователь может перейти к написанию рецензии (`ReviewScreen`) или просмотру рецензий других пользователей (`ReviewsScreen`), а также добавить трек в плейлист.

При нажатии на имя исполнителя открывается экран исполнителя (`ArtistCardScreen`) с его биографией, статистикой и списком треков.

На экране поиска (`SearchScreen`) пользователь может найти треки или исполнителей по названию, а также просматривать вертикальный фид рекомендаций с возможностью переключения между вкладками «Рекомендации» и «Для вас».

На экране оценок (`RatingsScreen`) пользователь видит все свои оценки треков с возможностью фильтрации и просмотра детальной статистики по пяти критериям.

На экране профиля (`ProfileScreen`) пользователь может редактировать никнейм, описание и аватар, а также просматривать свои любимые треки и исполнителей. Переход в настройки осуществляется через иконку в правом верхнем углу.

На экране настроек (`SettingsScreen`) пользователь может изменить тему оформления (тёмная, светлая, системная), настроить параметры воспроизведения, эквалайзер, таймер сна, а также выйти из аккаунта.
